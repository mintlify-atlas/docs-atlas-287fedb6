PBS TUI ships with four built-in themes. The `pbs-*` themes use a purpose-built color palette optimized for HPC dashboards, while the `ansi-*` themes use muted 256-color palette tones that blend with your terminal's own colorscheme. You can switch themes at any time without restarting the application.

## How to switch themes

**Via the command palette:**

1. Press `Ctrl+P` to open the command palette.
2. Type `Change theme` to filter the action list.
3. Select the theme you want from the results.

The change takes effect immediately without restarting the application. Your selection persists for the current session.

<Tip>
  The `ansi-*` themes use muted colors that are intentionally close to common terminal default palettes. If the colors in `pbs-dark` or `pbs-light` feel too saturated for your terminal setup, switching to `ansi-dark` or `ansi-light` produces a softer appearance.
</Tip>

## CSS customization

All layout rules and theme-specific attributes are defined in `pbs_tui/app.tcss`. If you need colors or spacing that none of the built-in themes provide, fork the project and edit that file directly. The Textual documentation covers the full CSS property set available to Textual applications.

<Note>
  Theme support requires Textual 0.47 or later, which introduced the `Theme` helper class used to register `pbs-dark`, `pbs-light`, `ansi-dark`, and `ansi-light`. Older Textual versions fall back to Textual's built-in default themes and the four PBS TUI themes will not appear in the command palette.
</Note>
