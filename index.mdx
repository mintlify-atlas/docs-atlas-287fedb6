PBS TUI is a terminal user interface built with [Textual](https://textual.textualize.io/) for monitoring [PBS Pro](https://altair.com/pbs-professional) schedulers. It surfaces job, queue, and node activity in a single live-updating dashboard — giving operators a real-time view of workload health without leaving the terminal.

## What you get

<CardGroup cols={2}>
  <Card title="Live PBS data" icon="arrow-rotate-right">
    Queries `qstat` and `pbsnodes` every 30 seconds, preferring JSON output with automatic XML and text fallbacks.
  </Card>
  <Card title="Cluster utilization grid" icon="grid-2">
    Visual map of node utilization across the whole cluster. Click any job block to inspect its details.
  </Card>
  <Card title="Rich data tables" icon="table-list">
    Sortable, filterable tables for jobs, nodes, and queues — each with a detail pane for the selected row.
  </Card>
  <Card title="Multiple themes" icon="palette">
    Ships with `pbs-dark`, `pbs-light`, `ansi-dark`, and `ansi-light` themes. Switch via the command palette.
  </Card>
  <Card title="Sample data mode" icon="flask">
    Bundled mock cluster (~560 nodes, ~40 jobs) lets you explore the UI without a live PBS scheduler.
  </Card>
  <Card title="Inline & headless modes" icon="terminal">
    Print snapshots non-interactively or run headless in CI — no interactive terminal required.
  </Card>
</CardGroup>

## Quick install

```bash
uv pip install pbs-tui
```

Then launch the dashboard once `qstat` and `pbsnodes` are on your `PATH`:

```bash
pbs-tui
```

<Tip>
  No PBS cluster? Try PBS TUI immediately with bundled sample data: `PBS_TUI_SAMPLE_DATA=1 pbs-tui`
</Tip>
