---
title: "Get started with PBS TUI"
description: "Install PBS TUI with pip or uv and launch the PBS Pro dashboard in under two minutes. Includes bundled sample data for trying the UI without a live cluster."
---

PBS TUI installs as a standard Python package and runs from a single command. Follow the steps below to go from zero to a live dashboard — or try the bundled sample data if you do not have a PBS cluster handy.

<Tip>
  Want a zero-install trial? Run `uvx pbs-tui` to launch PBS TUI directly using [uv](https://docs.astral.sh/uv/) without adding it to your environment.
</Tip>

<Steps>
  <Step title="Install PBS TUI">
    Install PBS TUI into your Python environment using `uv` or `pip`:

    <CodeGroup>

    ```bash uv
    uv pip install pbs-tui
    ```

    ```bash pip
    pip install pbs-tui
    ```

    </CodeGroup>

    PBS TUI requires Python 3.10 or later.
  </Step>

  <Step title="Try sample data">
    If you do not have a PBS Pro cluster available, launch PBS TUI with the bundled mock cluster:

    ```bash
    PBS_TUI_SAMPLE_DATA=1 pbs-tui
    ```

    This loads a realistic sample snapshot of approximately 560 nodes and 40 jobs so you can explore every part of the UI — jobs table, cluster grid, nodes, queues, and detail panel — without any scheduler connection. A warning banner appears at the bottom of the screen indicating that the data is synthetic.
  </Step>

  <Step title="Connect to your cluster">
    Once `qstat` and `pbsnodes` are on your `PATH`, launch the live dashboard:

    ```bash
    pbs-tui
    ```

    PBS TUI queries `qstat` and `pbsnodes` on startup and then refreshes automatically every 30 seconds. To use a different polling interval, pass `--refresh-interval` with a value in seconds:

    ```bash
    pbs-tui --refresh-interval 60
    ```

    If either PBS command is missing or returns an error, the status bar at the bottom of the screen shows the error message so you can see immediately when data is stale.
  </Step>

  <Step title="Navigate the dashboard">
    The dashboard is organized into five tabs: **Jobs**, **Cluster**, **Nodes**, **Queues**, and **Help**. Use the following key bindings to move around:

    | Key | Action |
    |:---:|:-------|
    | `q` | Quit the application |
    | `r` | Refresh data immediately |
    | `g` | Focus the Cluster tab |
    | `j` | Focus the Jobs tab |
    | `n` | Focus the Nodes tab |
    | `u` | Focus the Queues tab |
    | `d` | Toggle the detail panel |

    Selecting a row in any table opens the **detail panel** on the right with full information about that job, node, or queue. Press `d` to show or hide it. Use Tab and the arrow keys (or `PageUp`/`PageDown`) to move through rows once a table has focus.
  </Step>
</Steps>

<Note>
  For the full keyboard shortcut reference, see [Key bindings](/usage/key-bindings). To print a job table non-interactively without starting the TUI, see [Inline snapshot mode](/usage/inline-mode).
</Note>
